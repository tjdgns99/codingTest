시간복잡도
문제를 해결하기 위한 연산 횟수

시간복잡도 유형
빅오: 최악
1	1
logN	로그N
N	N
NlogN	N로그N
N^2	N제곱
2^N	2의 N승
n!	N팩토리얼

1번째 줄의 수의 개수 N(1<= N <=1,000,000), 오름 차순 정렬, 제한 시간 2초
제한 시간 2초 -> 2억번의 연산횟수
연산 횟수 = 알고리즘 시간 복잡도 X 데이터 크기

버블정렬 = N^2 = (1,000,000)^2 -> 2억보다 큼 -> 부적합
병합정렬 = NlogN = (1,000,000)log(1,000,000) -> 2억보다 작음 -> 적합

시간복잡도 
알맞은 알고리즘 선택 기준
비효율적인 로직 찾아서 효율적으로 바꾸기
